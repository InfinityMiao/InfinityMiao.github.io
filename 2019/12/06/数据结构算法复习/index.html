数据结构算法复习

2019-12-06

查找

查找表：用于查找的数据集合成为查找表

平均查找长度：ASL

线性结构

算法 思想 ASL 特性
顺序查找 从线性表的一段开始，逐个查找关键字是否满足条件 (n+1)/2
折半查找(二分查找) 递归地对处于mid位置的key进行比较，如果目标key在key的左面，则high=mid-1；否则low=mid+1；mid=⌊(low+high)/2⌋；如果目标key就是key，那么查找结束。 log(n+1)-1，最多是⌈log(n+1)⌉ 只能在有序的顺序存储表（不能是链表）中，可以将折半查找判定树形成一个高度为⌈log(n+1)⌉的平衡二叉树
分块查找（索引顺序查找） 将查找表分成多个块，块的内部是无序的，而块之间是有序的，即前一个块的所有关键字都小于后面块的关键字。每个块会记录块内的第一个关键字的地址，以供在块内进行顺序查找 有b个块，每个块有s个记录时，ASL=(b+1)/2+(s+1)/2 当s=√n时，ASL最小

树形结构

B树

  • 又称多路平衡查找树
  • 所有结点孩子的结点数的最大值成为B树的阶m，但是m阶的B树的子树最大值不要求是m
  • 根结点至少有2颗子树
  • 除了根结点外所有非叶节点至少有⌈m/2⌉颗子树，即⌈m/2⌉-1 <= 关键字个数 <= m-1个关键字
  • 高度不包括只有叶节点的那一层
  • 高度h的范围：logm(n+1)<=h<=log⌈m/2⌉((n+1)/2)+1
非叶节点结构
此节点包含的关键字树 子树的指针 第1个关键字 子树的指针 … 子树的指针 第n个关键字 子树的指针
n P0 K1 P1 … Pn-1 Kn Pn

其中Pi-1所指子树的所有关键字<Ki<其中Pi所指子树的所有关键字

B树插入
  1. 定位，使用查找算法找到插入该关键字的最底层中的某个非叶结点
  2. 插入，若插入后的关键字个数<=m-1，则直接插入；否则关键字数量超过m-1之后需要进行分裂
  3. 分裂，将新插入关键字的结点拆分为两部分，中间位置⌈m/2⌉的关键字上移至父节点。
B树删除
  1. 定位，使用查找算法找到插入该关键字的最底层中的某个非叶结点
  2. 删除，若删除后的关键字个数>=⌈m/2⌉-1，则直接删除；否则关键字数量小于⌈m/2⌉-1之后需要进行平衡
  3. 如果兄弟的关键字数量在借走一个之后仍然>=⌈m/2⌉-1，则将兄弟结点的父节点插入此节点中，将兄弟结点的第一个关键字取代其父节点
  4. 如果兄弟的关键字数量在借走一个之后<⌈m/2⌉-1，则将此节点与兄弟结点及其父节点进行合并

B+树

m阶与B+树和B树的差别：

  1. B+树种每个关键字对应一个子树
  2. B+树每个结点的关键字数量⌈m/2⌉<=n<=m，B树每个结点的关键字数量⌈m/2⌉-1<=n<=m-1，即B+树的关键字上限比B树+1
  3. 非叶结点仅是索引，不包括实际信息，叶节点包含信息
  4. B树不可以顺序查找，B+树可以
非叶节点结构
子树的指针 第1个关键字 … 子树的指针 第n个关键字
P1 K1 … Pn Kn

Ki<=其中Pi所指子树的所有关键字

叶节点结构

关键字信息的指针 第1个关键字 … 关键字信息的指针 第n个关键字 下一个叶节点指针
P1 K1 … Pn Kn P

散列结构

散列（哈希）函数：把查找表中的关键字映射成该关键字对应地址的函数。

散列（哈希）表：根据关键字而直接进行访问的数据结构，散列表建立了关键字和存储地址之间的一种直接映射关系。

哈希函数类型：

  • 直接定址法（线性）：H(key) = a x key + b

  • 除留余数法：H(key) = key % p

    ······

处理冲突的方法：

开放地址法

可存放新表项的空闲地址即向它的同义词表项开放，又向它的非同义词表项开放。

m为散列表表长

  • 线性探测法：di = 0，1，2，3…m-1。在冲突发生时，顺序查看表中的下一个单元是否为空，直到找到一个空闲单元。
  • 平方探测法：di = 0²，1²，-1²，2²，-2²…k²，-k²（K<=m/2)。在冲突发生时，在单元的左右两个方向查看表中查找空闲单元，增量平方上升。
  • 再散列法：di = Hash2(key)。在冲突发生时，使用第二个哈希函数计算该关键字的地址增量。
  • 伪随机序列法：di = 伪随机数列。在冲突发生时，使用随机数列作为增量。

拉链法

将所有的同义词存储在一个线性链表中，这个线性链表由其散列地址唯一标识。

查找效率

哈希表的查找效率取决于三个因素：

  • 哈希函数

  • 处理冲突的方法

  • 装填因子

    • 装填因子一般记为α，定义为一个表的装满程度
  • α = n/m，n为表中记录数，m为散列表长度
    • 平均查找长度依赖于α，而不会直接依赖于n和m

字符串匹配

串的模式匹配：求模式串在主串中的位置

简单的模式匹配算法

总体思想就是挨个挨个去匹配

最坏的时间复杂度为O(n x m)，n和m分别为主串和模式串的长度

KMP算法

在匹配的过程中，如果出现字符不等的情况，不需要回溯i指针，而是利用已经的到的“部分匹”的结果将模式向右滑动尽可能远的距离后，继续进行比较

next 数组各值的含义：代表当前字符之前的字符串中，有多大长度的相同前缀后缀。例如如果next [j] = k，代表a[1:k-1] = a[j-k:j-1]

求next数组

  1. next数组下标从1开始

  2. next[1] = 0，i = 1，j = 0

    • 如果j = 0，++i，++j，next[i] = j

    • 如果S[i] = S[j]，next[i] = j

  • 如果S[j-1] ≠ S[k]，j = next[j]

next数组示例

编号 1 2 3 4
模式串 a b a b
next 0 1 1 2

KMP匹配

  • 如果j=0，++i，++j
  • 如果S[i] = T[j]，++i，++j
  • 如果S[i] ≠ T[j]，i不变，j=next[j]
  • 如果j > 子串（模式串T）长度，最后返回i - 子串（模式串T）长度

时间复杂度O(n+m)
