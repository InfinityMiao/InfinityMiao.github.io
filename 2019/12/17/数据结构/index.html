数据结构

2019-12-17

线性表

顺序存储（物理结构）

特点：随机访问，可以在O(1)的时间内完成插入删除

链表（物理结构）

静态链表：以next = -1为其结束的标志

栈（逻辑结构）

栈顶指针（top）指向最后一个元素

顺序存储的栈空条件：S.top = -1

顺序存储的栈满条件：S.top = MaxSize -1

n种不同的元素进栈，有卡特兰数个出栈序列

队列（逻辑结构）

队头：允许删除的一端

队尾：允许插入的一端

顺序存储的队列

队头指针（front）指向第一个元素的地址

队尾指针（rear）指向最后一个元素的地址+1

队空条件：front = rear = 0

队满条件：rear = MaxSize

存储空间用完：front = rear = MaxSize

循环队列

队头指针（front）指向第一个元素的地址

队尾指针（rear）指向最后一个元素的地址+1

队空条件：Q.front = Q.rear

队满条件：（Q.rear+1)%MaxSize = Q.front

牺牲了一个存储单元，即可容纳的元素数量为n = MaxSize - 1

链式存储的队列

通常链式队列是没有头结点的单链表，即front指针直接指向第一个元素，rear指针指向最后一个元素

栈和队列的应用

栈

括号匹配

表达式求值

  • 中缀表达式==>后缀表达式，即将运算符放在操作数的后面
  • 使用后缀表达式进行表达式求值
    • 遇到操作数，将操作数进栈
    • 遇到操作符，pop两个操作数，进行运算后将结果push

队列

队列可以用于树的层次遍历

队列可以用于缓冲区以及CPU的资源调度

矩阵的压缩存储

二维数组按行优先顺序存放

二维数组按列优先顺序存放

稀疏矩阵可以使用三元组存储：i，j，v；i、j为坐标，v为值

树

一个结点的子结点个数为它的度

树中的结点最大的度为树的度

树的路径长度是从根结点到每个结点的路径长度总和

深度从根结点开始，高度从叶结点开始

二叉树

二叉树 ≠ 度为2的树，二叉树有序

完全二叉树

完全二叉树的前n-1层是一个满二叉树，有2^(n-1)-1个结点

完全二叉树的最后一层k层中从左到右

  • 先有双亲的度是2的叶子节点
  • 然后有一个或者没有双亲度是1的叶子节点
  • 剩下的为空

二叉树的存储

顺序存储：类似满二叉树的层次遍历的结果，0代表空

链式存储：lchild || data || rchild

二叉树遍历

二叉树的递归遍历转化为非递归

  • 初始化一个栈
  • 在p指针非空时
    • 遇到非空节点，将结点push
    • 根据遍历顺序决定什么时候pop，访问子树的根结点

先序和中序可以确定一个二叉树，中序和后序可以确定一个二叉树，即必须要有二叉树的中序遍历序列才能确定

线索二叉树

线索二叉树：ltag || lchild || data || rchild ||rtag

当tag为1时，lchild指向前驱，rchild指向后继

无法使用线索二叉树实现后序遍历的顺序，因为若非叶节点有右子树，那么无法返回到根结点

二叉树应用

二叉排序树（BSL）

删除

  • 若结点z只有左或右子树，则让z的子树代替z
  • 若结点z有左和右子树，则让z的中序后继代替z

ASL—-平均查找长度

平衡二叉树（BBL）

简称平衡树AVL

插入

  • LL：右单旋转
  • RR：左单旋转
  • LR：先左后右旋转
  • RL：先右后左旋转

查找

查找的次数不包括最后在叶节点与目标的比较

哈夫曼树

WPL：树中所有叶节点的带权路径长度之和，即带权路径长度

前缀编码：没有任何编码是其他编码的前缀

构造的哈夫曼编码不唯一

树

树的存储

双亲表示法：用数组存储所有结点，每个数据元素中包括结点值和双亲指针，根结点的双亲结点为-1

孩子表示法：用数组存储所有结点，每个数据元素中包括结点值和孩子的链指针，叶子结点的孩子结点为空

孩子兄弟表示法：firstChild || data || brother

树、森林转二叉树

树==>二叉树

类似于孩子兄弟表示法，左孩子右兄弟；左孩子表示为第一个孩子，右孩子表示第一个兄弟

根结点没有右孩子

森林==>二叉树

将森林的每一颗树转换为二叉树，将二叉树连接到第一棵树的右子树上

树和森林转二叉树画法

要以树根为轴心，顺时针旋转45°

树 森林 二叉树
先根遍历 先序遍历 先序遍历
后根遍历 中序遍历 中序遍历

图

弧尾—————->弧头

不存在重复的边，不存在顶点到自身的边的是简单图，否则就是多重图

完全图：任意两个顶点之间都有边

无向图

  • 连通：顶点v和w，v有路径到w

  • 连通图：若图中的任意两个顶点都是连通的，那么就是连通图

  • 连通分量：极大连通子图

极大：要求该连通子图包括其所有的边

极小：要求在保持连通的情况下，边数最小的子图

有向图

  • 强联通：顶点v和w，v有路径到w，w有路径到v
  • 强连通图：若图中的任意两个顶点都是强连通的，那么就是强连通图
  • 强联通分量：极大强连通子图

生成树、森林

非连通图的所有连通分量的生成树构成了树

入度，出度

带权图也称为网

简单路径：顶点不重复出现的路径

稀疏图：边数 < 顶点数 * log顶点数

图的存储

邻接矩阵

邻接表法

顶点表（所有的顶点）

顶点域 边表头指针
data firstarc（指向该顶点的第一个边）

边表（所有的边）

临接点域 指针域
adjvex（这条边的弧头） nextarc（指向这条边的弧尾的下一个边）

对于稀疏图，可以使用邻接表法节约空间

图的遍历

广度优先遍历（BFS）

广度优先遍历不是递归算法，因此需要辅助队列，而遍历一个连通子图的步骤基本和树的层次遍历一样

辅助队列保存所有顶点是否已经被遍历过，最后保证所有连通子图有调用过和层次遍历类似的遍历程序

广度优先遍历可以求单源最短路径

深度优先遍历（DFS）

深度优先遍历同样需要辅助队列保证所有连通子图被遍历到

图的应用

最小生成树

一个连通图的生成树是极小连通子图

最小生成树不是唯一的

最小生成树的边数是顶点数-1

Prim算法

边集E，顶点集V

算法思想：

  1. 从V中随机选一个顶点v1放入Vt
  2. 循环如下
    • 从E中选择一个尽可能小的边E1，边有一端在Vt中，另一端在V-Vt中
    • 将E1放入Et中，并将其端点放入Vt

Kruskal算法

边集E，顶点集V

算法思想：

  1. 从V中随机选一个顶点v1放入Vt
  2. 循环如下
    • 从E中选择一个尽可能小的边E1，要在加入最小生成树之后不构成回路

最短路径

Dijkstra算法

每趟确定一个顶点v的最短路径大小

确定一个顶点v最短路径大小之后，更新与他连通的顶点的最短路径，到下一趟

O（V³）

Floyd算法

第K步：更新路径表中，经过Vk到达的v的最短路径

拓补排序

有向无环图DAG

AOV网：用DAG成为顶点表示工程活动的网络

拓补排序：每趟输出图中入度为0的顶点，更新其他顶点的入度

AOE网：带权的AOV网，权值表示完成该活动的开销，边表示活动，而不是顶点

关键路径：具有最大路径长度的路径成为关键路径，关键路径上的活动称为关键活动
