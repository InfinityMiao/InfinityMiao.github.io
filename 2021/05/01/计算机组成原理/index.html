计算机系统概述

吞吐量：单位时间处理请求的数量

CPI：执行一条指令所需的时钟周期数

MIPS：每秒执行多少百万条指令

FLOPS：每秒执行多少次浮点运算

数据的表示和运算

数制与编码

进制转换

  • 二进制转八进制、十六进制：从小数点出发，以每3/4位为一维目标数进行转换

  • 二进制、八进制、十六进制转十进制：按权展开相加法

  • 十进制转其他进制：

    • 整数部分除基（2/8/16）取余，低位->高位

    • 小数部分乘基（2/8/16）取余，高位->低位

    • 并不是所有十进制小数都可以用二进制表示

编码

真值

  • 符号位为0表示正数，符号位为1表示负数

BCD码

  • 8421码：每四位二进制码表示一位十进制码，0000-1001表示0-9，无法表示10-16。当两个8421码相加大于9（1001）时，需要加6（0110）修正即用8位的二进制码表示2位十进制码
  • 余三码：8421码+(0011)~2~，即每个数加三
  • 2421码：权值由高到低分别没2、4、2、1，因此最大的(1111)~2~只能表示2+4+2+1=9

校验码

  • 奇偶校验码
    • 奇校验码：在校验位加上0或1后，整个校验码中“1”的个数为奇数
    • 偶校验码：在校验位加上0或1后，整个校验码中“1”的个数为偶数

  • 校验位一般在最高位

  • 海明校验码

    • 海明码的位数：2^k^-1≥n+k（n为信息位位数，k为校验位位数）

    • 海明码的位置：第i位海明码在2^i-1^的位置上，即P1在H1，P2在H2，P3在H4，P4在H8

    • 海明码的校验位：

      • 将信息位的位置序号用k位二进制数表示出来

      • 校验位Pi校验位置序号的第i位为1的信息位

      • 例：
        | 信息位位置 | 第3位位置序号 | 第2位位置序号 | 第1位位置序号 |
        | ————— | ——————- | ——————- | ——————- |
        | H3 | 0 | 1 | 1 |
        | H5 | 1 | 0 | 1 |
        | H6 | 1 | 1 | 0 |
        | H7 | 1 | 1 | 1 |
        | | P3 | P2 | P1 |

        故校验位P1对H3、H5、H7进行偶校验

        校验位P2对H3、H6、H7进行偶校验

        校验位P3对H5、H6、H7进行偶校验

      • 海明码的纠错

        • S1对P1、H3、H5、H7进行偶校验
        • S2对P2H3、H6、H7进行偶校验
        • S3对P3、H5、H6、H7进行偶校验
        • 当S3S2S1=101时，即说明H5出错，当S3S2S1=001时，即说明H1出错

    • 全局纠错码：全局纠错码对所有信息位与校验位进行偶校验

      • 纠错能力：能够检测2位，纠错1位

  • CRC校验码（循环冗余校验码）

    • 校验码的位数：2^k^-1≥n+k（n为信息位位数，k为校验位位数）
    • 校验码的位置：在信息位的后面，即：| 信息位 | 校验位 |

    • 校验码的生成

      • 将原信息码逻辑左移k位

      • 对移位后的信息码用生成多项式G(x)进行模2除，其余数即为CRC校验码

    • 检测和纠错：当CRC码（| 信息位 | 校验位 |）对生成多项式G(x)进行模2除，若余数为0则正确，非0否则出错。CRC没有纠错能力
    • 生成多项式为除数：例G(x)=x^3^+x+1对应于1011
    • 模二除：对比被除数与除数的最高位，决定结果为1还是0
    • 异或：相同为0，相异为1

定点数的表示与运算

定点数

  • 定点整数表示范围：-(2^n^-1)~2^n^-1
    • 定点小数表示范围：-(1-2^-n^)~1-2^-n^

原码、反码、补码、移码

正数 负数
原码 符号位为0 符号位为1
反码 [x]~反~=[x]~原~ 原码的符号位不变，数值部分取反
补码 [x]~补~=[x]~原~ 原码的符号位不变，数值部分取反，并在末位加1，即[x]~补~=[x]~反~+1
移码 [x]~移~=2^n^+x [x]~移~=2^n^+x，其中机器字长为n+1

移位

  • 算数移位
正数 负数
原码 添0 添0
反码 添0 添1
补码 添0 左边添1，右边添0
  • 逻辑移位：不管是左移还是右移都添0
  • 循环移位：分为带进位与不带进位的循环左移与循环右移

加减法

  • 补码定点数加减法
    • 符号位与数值位一起参加运算，符号位运算产生的进位要丢掉，结果的符号由运算得出
    • [A+B]~补~=[A]~补~+[B]~补~
    • [A-B]~补~=[A]~补~+[-B]~补~
    • 补码运算的结果亦为补码，补码转原码：
      • 正数——不变
      • 负数——第2位~第n-1位取反
  • 溢出的判定
    • 采用一位符号位
      • 设A的符号位为A~s~，B的符号位为B~s~，运算结果的符号位为S~s~
      • 若A~s~=B~s~=0，S~s~=1，则上溢出
      • 若A~s~=B~s~=1，S~s~=0，则下溢出
      • 若A~s~=B~s~=S~s~=1，或A~s~=B~s~=S~s~=0，则不溢出
      • 若A~s~≠B~s~，则不溢出
    • 采用双符号位
      • 若运算结果的两个符号位S~s1~S~s2~相同，则未溢出
      • 若运算结果的两个符号位S~s1~S~s2~不同，则溢出
      • S~s1~S~s2~=00，或S~s1~S~s2~=11未溢出
      • S~s1~S~s2~=01，上溢出
      • S~s1~S~s2~=10，下溢出
    • 采用进位判断
      • 若进位C~s~与最高数位C~1~相同，则未溢出，否则发生溢出

乘法

  • 原码一位乘法

    • 符号位异或（偶校验）：X~s~⊕Y~s~
    • 乘数的符号位不参与运算
    • 原码乘法用逻辑右移
    • 每一步对ACC里的乘积高位逻辑右移，多余的数移至MQ的乘积低位中，乘积低位抛弃掉最后一位，直到乘数全部被丢弃

  • 补码一位乘法

    • 符号位异或（偶校验）：X~s~⊕Y~s~
    • 乘数的一位符号位参与运算
    • 补码乘法用算数右移，即正数添0，负数添1
    • 每一步对ACC里的乘积高位算数右移，多余的数移至MQ的乘积低位中，乘积低位抛弃掉最后一位，直到乘数的数值位全部被丢弃
    • 辅助位默认为0
      • 辅助位-MQ最低位=0，ACC+0
      • 辅助位-MQ最低位=1，ACC+[x]~补~
      • 辅助位-MQ最低位=-1，ACC+[-x]~补~

  • | | 符号位 | 累加次数 | 移位 |
    | —————— | ———————————————————————— | ———— | ———— |
    | 原码一位乘法 | 不参与运算，乘数中有0位符号位，被乘数有2位符号位 | n | 逻辑移位 |
    | 补码一位乘法 | 参与运算，乘数中有1位符号位，被乘数有2位符号位 | n+1 | 算数移位 |

  • 原码、补码正数：尾数的数值第一位必须为1

  2. 尾数求和

IEEE-754

结构：|数符s|阶码的数值部分E（用移码的变种表示）|尾数M（用原码的变种表示）|

其中阶码的数值部分为[x]~移~=2^n^-1+x，其中机器字长为n+1，2^n^-1为偏置值

尾数的最高位1默认隐藏

规格化的浮点数真值=(-1)^s^x1.M^{E-偏置值}^，偏置值=2^n^-1

ALU

全加器

串行加法器：只有一个全加器，各个操作数串行地相加。成本低，速度慢

并行加法器：多个全加器同时对各位进行相加，但是低位的进位会影响到高位。因此并行加法器的速度被进位的产生与传递速度所限制

并行加法器有串行进位与并行进位之分：

  • 串行进位
  • 并行进位：同时产生进位，但是位数过多时会使表达式过长，可以采用分组并行进位
  • 分组并行进位：将n位全加器分为若干个小组，小组内的各位之间并行进位，小组与小组之间可以串行，也可以并行
    • 单级并行进位：组内并行，组间串行
    • 多级并行进位：组内并行，组间并行

双端口RAM和多模块存储器

    • 双端口RAM在同时写同一地址、同时读和写同一地址时会发生冲突
    • 可以增加标志位，作为开关
  2. 单体多字存储器
    • 数据线宽度与存储字长相同
  3. 多体并行存储器
    1. 高位交叉编址（顺序方式）
      • 体号在高位
      • 不能提高存储器吞吐率
    2. 低位交叉编址（交叉存储器）
      • 体号在底位
      • 顺序读写时，可以提高存储器吞吐率
      • r为总线传送周期（读写一个存储单元的数据时，数据总线的占用时间，可以理解为存取时间）
      • T为模块的存取周期，T=r+恢复时间，每经过r时间延迟后启动下一个模块
      • m≥T/r，使某模块在mxr时间后再次启动该模块时，上次存取操作已经完成
      • 连续读写n个字所需时间t=T+(n-1)r

Cache的基本工作原理

  • Cache与主存的数据交换以Cache块为单位，Cache与CPU的数据交换以字为单位
  • 命中率$H=\frac{访问Cache次数}{访问Cache次数+访问主存次数}$
  • 平均访问时间T~a~=Ht~c~+(1-H)t~m~，t~c~为Cache的访问时间，t~m~为主存的访问时间
  • 系统的效率e=t~c~/T~a~，即系统的效率为$\frac{只有Cache的访问时间}{Cache-主存体系的访问时间}$

Cache替换算法

  • 随机算法
  • 先进先出算法（FIFO）
  • 近期最少使用算法（LRU）：局部的
  • 最不经常使用算法（LFU）：全局的
  • 做题：用表格法

Cache写策略

  • 对Cache写命中
    • 全写法：同时在Cache和主存写，并在Cache与主存之间建立一个==写缓冲==
    • 写回法：先在Cache写，并在脏位（标记位）置1，表示此块被CPU修改过。当此块换出时，若脏位为1，则对主存写一遍
  • 对Cache写不命中（内容不在Cache中，而在主存中）
    • 写分配法：先把主存中的块调入Cache，再在Cache中修改
    • 非写分配法：只在主存中修改，不再调入Cache
  • 写回法与写分配法配合（以Cache为主），全写法与非写分配法配合（以主存为主）

加快地址转换：快表（TLB）

  • 快表由高速缓冲器件组成，慢表储存在主存中

  • 在地址转换时，先查找快表，若命中则无需访问主存中的页表（慢表）

  • 缺页处理：即信息不再主存中，则主存从外存中读取一页，并更新页表和TLB

  • Cache缺失处理：即信息不再Cache中，则Cache从主存中读取一块，并更新有效位

段式虚拟存储器

  • 段式虚拟存储器的段是按程序的逻辑结构划分的
  • 段表（在主存中）：包括段首址、段长、装入位
  • 虚拟地址：包括|段号|段内地址|，根据段号查找这一段在段表中的哪一行，进而确定段首址，段首址与段内地址（偏移量）相加得到对应的主存实地址

段页式虚拟存储器

程序按逻辑结构分段，每段再划分为大小固定的页

指令系统

指令格式

指令是计算机运行的最小功能单位

一台计算机的所有指令的集合构成该机的指令系统（指令集）

指令的基本格式

  • 指令包括了：操作码字段、地址码字段
  • 零地址指令：例如中断、停机等指令
  • 一地址指令：OP(A~1~)->A~1~/(ACC)OP(A~1~)->ACC，例如求补码、反码等
  • 二地址指令：(A~1~)OP(A~2~)->A~1~
  • 三地址指令：(A~1~)OP(A~2~)->A~3~
  • 四地址指令：(A~1~)OP(A~2~)->A~3~，A~4~为下一条要执行的指令的地址

扩展操作码

例如，长度为4的操作码只有2^4^-1个，第2^4^个是1111，仅表示操作码要扩展，地址码要减少。

长度为8的操作码只有2^4^-1个，第2^4^个是1111 1111，仅表示操作码要扩展，地址码要减少。

……

  • 不允许短码是长码的前缀，即操作码是前缀码
  • 不同指令的操作码不能重复
  • 使用频率较高的指令分配较短的操作码，频率较低的指令分配较长的操作码，类似于哈夫曼树

指令的寻址方式

指令寻址

  • 顺序寻址：通过PC自增+1实现
  • 跳跃寻址：通过转移类指令实现。通过相对或绝对地址得到下一条指令的地址，并据此修改PC值，故下一条指令的地址仍由PC给出

数据寻址（指令的操作数）

  • 指令形式：|操作码|寻址特征|形式地址A|

  • 定义：真实地址EA，形式地址A
    | | 有效地址 | 访存次数 | |
    | ——————— | —————- | ———— | ————————————————————————— |
    | 隐含寻址 | 程序指定 | 0 | 指令中隐含操作数的地址 |
    | 立即寻址 | A既是操作数 | 0 | A是操作数本身 |
    | 直接寻址 | EA=A | 1 | A是操作数的真实地址EA |
    | 间接寻址 | EA=(A) | 2 | A是操作数地址的地址 |
    | 寄存器寻址 | EA=R~i~ | 0 | A是保存着操作数地址的寄存器的编号 |
    | 寄存器间接寻址 | EA=(R~i~) | 1 | A是保存着操作数的地址的地址的寄存器的编号 |
    | 相对寻址 | EA=(PC)+A | 1 | A是操作数地址相对于==下一条指令==地址的位移量 |
    | 基址寻址 | EA=(BR)+A | 1 | A+基址寄存器的内容即为操作数在主存中的位置 |
    | 变址寻址 | EA=(IX)+A | 1 | A+变址寄存器的内容即为操作数在主存中的位置 |
    | 堆栈寻址 | | | 在堆栈计算机中，零地址运算类指令的操作数就在堆栈中 |

  • | 基址寻址 | 变址寻址 |
    | —————————————————————————————— | ————————————————————————— |
    | 基址寄存器可用专用寄存器或通用寄存器 | 变址寄存器可用通用寄存器 |
    | 基址寄存器中的内容在程序运行的过程中不可改变 | 变址寄存器中的内容在程序运行的过程中可以改变 |
    | 基址寄存器面向系统，基址寻址用于为多道程序或数据分配存储空间 | 变址寄存器面向用户，变址寻址用于处理数组、循环问题 |

  • 当采用通用寄存器R~0~为基址寄存器时，则指令形式为|操作数|寻址特征|通用寄存器R~0~地址|形式地址A|

中央处理器

CPU的功能和基本结构

  • CPU的具体功能

    • 指令控制：完成取指令、分析指令、执行指令的操作
    • 操作控制：管理并产生每条指令的操作信号，并将其送往相应的部件，从而控制这些部件按指令进行
    • 时间控制：对各种操作加上时间上的控制
    • 数据加工：对数据进行算数和逻辑运算，有CPU的运算器完成
    • 中断处理：对异常情况进行处理

  • 一条指令的功能往往由若干操作信号的组合来实现

    | 运算器 | 控制器 |
    | ———————————————————————————- | —————————————————————————————— |
    | 算数逻辑单元（ALU） | 程序计数器（PC）：指出下条指令在主存中的地址 |
    | 通用寄存器 | 指令寄存器（IR）：保存当前指令 |
    | 累加寄存器（ACC）：暂存ALU的结果 | 指令译码器：对操作码字段进行译码，为控制器提供特定操作信号 |
    | 暂存寄存器：暂存从主存中读来的数据 | 存储器地址寄存器（MAR）：存放要访问的主存单元的地址 |
    | 程序状态寄存器（PSW）：保留运算指令或测试指令的状态信息 | 存储器数据寄存器（MDR）：存放读出或要写入的数据 |
    | 移位器：对操作数或运算结果进行移位运算 | 时序系统：产生各种时序信号 |
    | 计数器（CT）：控制乘除运算的操作步数 | 微操作信号发生器：根据IR中的指令和PSW的内容和时序信号，产生各部件要的控制信号 |

指令执行过程

  • CPU从主存中取出并执行一条指令的时间为指令周期

  • 指令周期由若干机器周期组成，机器周期包含若干时钟周期（也称节拍或T周期它是CPU操作的基本单位）

  • 机器周期以存取周期为基准时间

  • 一个完整的指令周期可以分为4个周期

    | | 取指周期 | 间址周期 | 执行周期 | 中断周期 |
    | —— | ——————————————————— | ————————— | —————————————————————————- | —————————————————————————— |
    | 目的 | 根据PC的内容从主存中取指令代码放到IR中 | 取操作数的有效地址 | 根据IR中的指令字的操作码和操作数通过ALU操作产生结果 | 处理中断请求，==只有出现中断请求时，才会有中断周期== |

  • 取指周期

    1. PC->MAR->地址总线->主存：将PC中的指令地址给主存
    2. CU->控制总线->主存：CU发出控制信号，让主存进行传指
    3. 主存->数据总线->MDR->IR：主存将指令传到MDR，IR在MDR取指
    4. CU->PC：CU让PC+1

  • 间址周期

    1. IR->MAR->地址总线->主存：将IR中操作数的虚拟地址给主存
    2. CU->控制总线->主存：CU发出控制信号，让主存进行数据寻址
    3. 主存->数据总线->MDR：主存将数据的有效地址传到MDR

  • 执行周期

    执行周期没有统一的数据流向

  • 中断周期（假设将程序断点存入堆栈中，SP指示栈顶地址）

    1. CU->MAR->地址总线->主存：CU控制将SP减1，并将栈顶地址传至主存
    2. CU->控制总线->主存：CU发出写指令
    3. PC->MDR->数据总线->主存：将程序断点存入主存
    4. CU->PC：CU让PC指向下一个指令的地址

  • 指令执行方案

    • 单指令周期：所有指令都选用相同的执行时间完成，指令之间串行
    • 多指令周期：不同类型的指令选用不同的执行步骤（时间）完成，指令之间串行
    • 流水线周期：指令之间可以并行执行的方案，通过在每个时钟周期启动一条指令，尽量让多条指令并行，但各自处在不同执行步骤中

数据通路

  • 数据在功能部件之间传送的路径称为数据通路，路径上的部件称为数据通路部件。

  • 数据通路实现了CPU内部的运算器与寄存器和寄存器之间的数据交换

  • 内部总线与系统总线

    • 内部总线：CPU内部连接运算器与各寄存器的总线
    • 系统总线：同一台计算机连接各部件的总线，例如CPU、内存、各类I/O接口间互相连接的总线

  • 数据通路的基本结构

    | 内部总线 | CPU内部单总线方式 | CPU内部三总线方式 | 专用数据通路方式 |
    | ———— | ———————————————————————— | ———————————————————————— | —————————————————————————————— |
    | | 将所有寄存器的输入和输出端都连接到一条公共通路上 | 将所有寄存器的输入和输出端都连接到多条公共通路上 | 根据指令执行过程中的数据和地址的流动方向安排连接线路，避免使用共享总线 |

  • 数据传送

    • 寄存器之间的数据传送：例如PC和MAR之间
    • CPU与主存的数据传送：例如间址周期，从主存中取指令
    • 执行算数或逻辑运算：先将一个操作数经过总线放在ALU的暂存器中，并在ALU的其中一端始终有效，然后将另一个操作数经过总线送入ALU的另一端，实现同时送入两个操作数

控制器

  • 控制器部件从数据总线接收指令信息，从运算器部件接收指令转移地址，送出指令地址到地址总线，并从控制总线向系统中各部件传送控制信号

  • 控制器的主要功能：

    • 从主存中取出一条指令，存储到IR中，并更改PC使其指出下一条指令在主存中的位置
    • 对指令进行译码或测试，产生相应的控制信号，以便启动规定的动作
    • 指挥并控制CPU、主存、输入输出设备之间的数据流动方向

  • 根据控制器产生微操作控制信号方式的不同，控制器可分为硬布线控制器、微程序控制器

    这两种控制器确定和表示指令执行步骤的办法以及给出控制信号的方案是不同的

  • | | 硬布线控制器 | 微程序控制器 |
    | ———— | —————————————————————————————— | —————————————————————————————— |
    | 工作原理 | 微操作控制信号由组合逻辑电路根据当前的指令码、状态和时序，即时产生 | 微操作控制信号以微程序的形式放在控制存储器中，执行指令时读出即可 |
    | 执行速度 | 快 | 慢 |
    | 规整性 | 繁琐，不规整 | 较规整 |
    | 应用场合 | RISC（ARM） | CISC（X86） |
    | 易扩充性 | 困难 | 易扩充 |

  • 硬布线控制器设计步骤

    1. 列出微操作命令的操作时间表，包括每个机器周期、节拍下的每条指令完成的微操作控制信号

    2. 进行微操作信号的综合

      将节拍提到最外面，周期次之，最里面是微操作，例：节拍1{取指(微操作1+微操作2)+间指(微操作1+微操作2)}+节拍2{取指(微操作1+微操作2)}

    3. 画出微操作的逻辑图

  • | 名词 | 释义 |
    | ——— | —————————————————————————————— |
    | 微程序 | 微程序设计的思想就是将每条机器指令编写成一个微程序，这些微程序存储在一个控制存储器中 |
    | 微指令 | 每个微程序包括若干微指令 |
    | 微操作 | 每条微指令操作对应一个或几个微操作命令，这些微操作是计算机中最基本、不可再分的操作。 |
    | 微命令 | 向执行部件发出的各种控制命令称为微命令，是控制序列的最小单位 |
    | 微周期 | 从控制存储器中读取一条微指令并执行相应的微操作所需时间 |

  • 微程序控制器结构

    | 寄存器 | 作用 |
    | ———————— | ————————————————- |
    | CMAR | 存放控制存储器的读/写微指令的地址 |
    | CMDR/μIR | 存放从控制存储器中读取的微指令 |
    | 控制存储器（CM） | 存放微程序，在CPU内部，用ROM实现 |

  • 一般来说取指令的微程序是公共的

  • 微程序控制器工作过程

    1. 机器自动取指：将取指微程序的入口地址放在CMAR并执行，从CM取出指令放在CMDR，然后在主存中取指令放在IR
    2. 机器指令翻译：将IR中的指令通过”微地址形成部件”，获得此周期的微指令入口地址，并送入CMAR
    3. 执行微指令：从CM中逐条取出对应的微指令并执行
    4. 继续下一条机器指令：返回到第2步继续执行，直到程序执行完毕

  • 微指令组成

    | 操作控制字段（微操作码字段） | 顺序控制字段（微地址码字段） |
    | —————————————— | —————————————— |
    | 用于产生控制信号 | 用于产生下一步微指令地址 |

  • 微指令编码方式（也称控制方式）

    • 直接译码：选用或不选用某个微命令，直接将该微命令对应的位设置为1或0即可，每位对应一个微操作

    • 字段直接译码：微命令字段拆分为若干小字段，互斥性微命令在同一字段，相容性微命令在不同字段

      每个字段独立编码，并代表一个微指令

      每段要留出一个状态位，表示本段不发出任何微指令

    • 字段间接编码

  • 微指令的格式

    | | 水平型微指令 | 垂直型微指令 |
    | ——————————— | ————————————————— | ———————————————— |
    | 原理 | 每条指令包含多个操作，采取直接译码 | 每条指令只包含一个操作，采取编码 |
    | 操作能力，效率，灵活性 | 强 | 弱 |
    | 微指令长度 | 长 | 短 |
    | 微程序长度 | 短 | 长 |
    | 掌握难度 | 高 | 低 |

  • 微程序控制器设计步骤

    • 写出对应机器指令的微操作命令及节拍安排
      1. 先确定不包括读微指令的微命令（类似于硬布线的微命令）
      2. 在每个微命令之间添加Ad(CMDR)->CMAR，即将下一个微命令的地址移入CMAR
    • 确定微指令格式
    • 编写微指令码点

流水线

  • 一条指令的执行过程可以分为三个阶段

    1. 取指
    2. 分析：包括译码和取操作数
    3. 执行

  • 指令在CPU的执行方式

    | t为微周期 | 顺序执行方式 | 一次重叠执行方式 | 二次重叠执行方式 |
    | ————- | ——————————————— | ————————————————— | ————————————————— |
    | 工作原理 | 每执行完一条指令，才取指下一条 | 每分析完一条指令，就开始取指下一条 | 每取指完一条指令，就开始取指下一条 |
    | 时间 | 3nt | (1+2n)t | (2+n)t |

  • 流水线的分类方式

    • 根据使用级别分类：部件功能级，处理机级，处理机间级
    • 根据完成的功能分类：单功能，多功能
    • 根据各段的连接方式分类：静态，动态
    • 根据各功能段是否有反馈信号分类：线性，非线性

  • 动态多功能流水线和静态多功能流水线

    • 单功能流水线：只能实现一种固定功能的流水线
    • 多功能流水线：通过各段间的不同连接方式，可以实现多种功能的流水线
    • 动态流水线指在同一时间内，流水线各段只能按同一种功能的连接方式工作
    • 静态流水线指在同一时间内，流水线各段的能按不同的运算工作

  • 流水线的性能指标

    | | 吞吐率 | 加速比 | 效率 |
    | —— | ———————————————— | —————————————————— | ———————————————- |
    | 原理 | 单位时间内，流水线完成的任务数量 | 不使用流水线与使用流水线所用的时间比 | 流水线的设备利用率 |
    | 计算 | 任务数量/时间 | 不使用流水线的时间/使用流水线的时间 | 时空图的有效面积/时空图的总面积 |

  • 超标量流水线：一个时钟周期内，通过内置多条流水线实现同时执行多个指令

  • 超流水线：在一个时钟周期内在分段，使一个功能部件在一个时钟周期内使用多次

  • 影响流水线的因素

    • 数据冲突（数据冒险）：下一条指令会用到上一条指令计算出的结果，此即数据冒险。多条指令重叠处理时会发生冲突，解决办法如下
      1. 遇到数据相关的指令及其后续指令都暂停一至几个时钟周期，直到数据相关问题消失
      2. 设置相关专用通路，即在前一条指令不会将结果写入寄存器组，下一条指令也不会读取寄存器组的数据，而是直接将上一条的结果作为输入数据执行。也成为==数据旁路技术==
      3. 编译优化，调整指令顺序解决
    • 控制冲突（控制冒险）：一条指令要确定下一条指令的地址。例如强制转移改变PC值，这会造成断流，解决方法如下
      1. ==分支预测==：分为简单（静态）预测和复杂（动态）预测
      2. 预取转移成功和不成功的两个控制流方向上的目标指令
      3. 加快和提前形成条件吗
      4. 提高转移方向的猜准率

总线

总线概述

  • 总线按功能分类：

    • 片内总线：CPU内部的总线
    • 系统总线：连接CPU、主存、I/O接口的总线
      • 地址总线
      • 控制总线
      • 数据总线
    • 通信总线：不同计算机系统之间传送信息的总线，例如网线

  • 总线按时序控制方式可分为：同步总线、异步总线

  • 总线按数据传输格式可分为：并行总线，串行总线

  • 总线的结构

    • 单总线结构：CPU、主存、I/O设备都挂在一组总线上
    • 双总线结构：CPU、主存在主存总线上，所有I/O设备在I/O总线上，两条总线通过桥接器连接
    • 三总线结构：CPU、主存在主存总线上，所有I/O设备在I/O总线上，I/O总线同时与CPU相连，高速I/O设备（例如磁盘）与主存通过DMA总线连接

  • 总线的性能指标

    | 名称 | 释义 |
    | ———— | —————————————————————————————— |
    | 传输周期 | 指一次总线操作时所需的时间，包括申请、寻址、传输、结束四个阶段 |
    | 工作频率 | 总线周期的倒数 |
    | 时钟周期 | 即机器的时钟周期 |
    | 时钟频率 | 即机器的时钟频率 |
    | 总线宽度 | 总线位宽，通常指==数据总线==的根数 |
    | 总线带宽 | 总线的数据传输率，总线带宽=总线工作频率(总线宽度\8) |
    | 总线复用 | 指一种信号线在不同时间传输不同的信息 |
    | 信号线数 | 地址总线、数据总线、控制总线三种总线的总线数和 |

总线仲裁

  • 为解决多个主设备同时竞争总线控制权的问题，可以采用总线仲裁部件，以某种方式确定一个主设备优先获得总线控制权

  • 按照是否将控制逻辑集中于一个设备（如CPU）中，可以将仲裁方式分为集中仲裁和分布仲裁

  • 集中仲裁

    设备在获得控制权后，通过总线忙信号线，向其他设备通知总线已被占用

    | 名称 | 工作原理 | 控制线数 |
    | ——————— | —————————————————————————————— | —————- |
    | 链式查询 | 主设备发送请求，若空闲，则总线控制器将响应信号通过响应线BG串行的向所有主设备传输。
    主设备依次接收到响应信号后，若其正在请求，则成功获得总线控制权，并且再次将信号截住。
    因此，距离控制器越近的设备，优先级越高 | 3 |
    | 计数器定时查询 | 总线控制器中有计数器。
    主设备发送请求，若空闲，则总线控制器将其计数器中的计数值通过⌈log~2~n⌉条的BG线（总线允许线）发向各个部件。
    当主设备的地址与计数值相同时，设备获得总线控制权。
    若没有主设备的地址与计数值相同且正在请求总线，则计数值自增。这是一种循环的方法 | ⌈log~2~n⌉+2 |
    | 独立请求方式 | 每个设备都有与控制器相连的请求线BR与允许线BG。
    当部件需要总线时，通过BR向控制器发送请求，此请求在控制器中排队。
    当轮到一个部件时，控制器通过BG向设备发送允许信号并获得控制权。 | 2n+1 |

  • 分布仲裁

    • 每个主设备都有各自的仲裁号和仲裁器
    • 设备需要总线时，将自己的仲裁号与仲裁总线上的仲裁号进行比较，若自己的优先级更高，则获得总线控制权
    • 当设备获得总线控制权后，用自己的仲裁号覆盖仲裁总线

总线操作和定时

  • 总线传输的四个阶段

    1. 申请分配阶段：设备提出申请，进行仲裁
    2. 寻址阶段：获得使用权的设备发出从模块的地址及有关命令，启动从模块
    3. 传输阶段：主模块与从模块进行数据交换，可以单向或双向
    4. 结束阶段：主模块从总线上撤除信息，让出使用权

  • 同步定时方式

    指系统采用统一的时钟信号协调发送和接受方的传送定时关系

  • 异步定时方式

    完全依靠传送双方的“握手”信号来实现定时控制

    根据“请求”和“回答”信号的撤销是否互锁，异步定时方式可以分为以下三种类型

    • 不互锁方式：
      • 主设备发出“请求”后，无需等待从设备的“回答”信号，在经过一段时间后自动撤销“请求”信号
      • 从设备在收到“请求”后，发出“回答”一段时间，也自动撤销“回答”信号
    • 半互锁方式：
      • 主设备发出“请求”后，必须等待从设备的“回答”信号，并撤销“请求”信号
      • 从设备在收到“请求”后，发出“回答”信号，无需等待主设备撤销“请求”，一段时间后自动撤销“回答”信号，
    • 全互锁方式
      • 主设备发出“请求”后，必须等待从设备的“回答”信号，并撤销“请求”信号
      • 从设备收到“请求”后，发出“回答”信号，必须等待主设备撤销“请求”，然后并撤销“回答”信号

输入/输出系统

I/O系统基本概念

  • I/O软件：包括驱动程序、用户程序等。通常用I/O指令和通道指令实现CPU与I/O的信息交换
  • I/O硬件：包括外设、设备控制器、接口、I/O总线等。通过设备控制器来控制I/O设备的具体动作；通过I/O接口与主机（总线）相连

I/O接口

  • I/O接口的功能
    • 实现主机与外设的同行联络控制：协调主机与外设不同的工作速度
    • 进行地址译码和设备选择
    • 实现数据缓冲：接口设置数据缓冲寄存器，用以数据的暂存
    • 信号格式的转换
    • 传送控制命令和状态信息
  • I/O端口是指I/O接口中用于缓冲信息的寄存器
  • I/O端口的编制方式
    • 统一编制：又称存储器映射方式，即将I/O端口当做存储器的单元进行地址分配，用统一的访存指令就可以访问I/O端口
    • 独立编制：又称I/O映射方式，需要设置专门的I/O指令来访问端口

I/O控制方式

  • 程序查询方式：CPU不断查询设备的状态信息是否做好准备，做好准备后，进行传输，然后修改传送参数，判断是否传输完毕

  • 程序中断方式

    • 程序中断：计算机在执行现行程序时，出现某些急需处理的异常情况时，CPU暂停现行程序，转而处理这些异常情况，处理完毕后CPU又返回到现行程序断点处，继续执行原程序

    • 外中断：来自CPU和内存以外设备引起的中断

    • 内中断：来自CPU和内存引起的中断

    • CPU响应中断条件

      1. 中断源有中断请求
      2. CPU允许进行中断和开中断
      3. 一条指令执行完了，且没有优先级更高的任务

    • 中断隐指令（响应过程）

      CPU在响应中断，需要进行一系列的操作才能去执行中断服务程序。这一系列操作是由硬件直接实现的，被称为隐指令，具体操作如下

    • 中断向量（入口地址）

      不同设备有不同的中断服务程序，每个中断服务程序有其各自的入口地址，即中断向量

      存储中断向量的区域叫做中断向量表

      中断向量地址即指中断服务程序的入口地址的地址

    • 中断处理过程

      | 过程 | 工作原理 | 实现对象 |
      | ———————— | —————————————————————————————— | :—————- |
      | 关中断 | 在中断服务程序前，关闭中断响应，保护中断现场，使其不能被新的中断所打断 | 硬件完成 |
      | 保存断点 | 确保完成中断服务程序后能够正确的返回到原程序，将==PC==（程序计数器）的内容保护起来 | 硬件完成 |
      | 引出中断服务程序 | 取出中断服务程序的入口地址并传送给PC | 硬件完成 |
      | 保护现场和屏蔽字 | 保存程序状态字、中断屏蔽寄存器和CPU中某些寄存器的内容 | 中断程序完成 |
      | 开中断 | 允许更高级中断请求得到相应，实现中断嵌套（多重中断） | 中断程序完成 |
      | 执行中断服务程序 | | 中断程序完成 |
      | 关中断 | 保证在恢复现场和屏蔽字时不被中断 | 中断程序完成 |
      | 恢复现场和屏蔽字 | | 中断程序完成 |
      | 开中断 | 恢复现场后，允许中断 | 中断程序完成 |
      | 中断返回 | 返回到原程序的断电处，以便继续执行原程序 | 中断程序完成 |

    • 多重中断（中断嵌套）：在中断处理时，有一个开中断时期，此时中断服务程序还没有正式执行，如果此时有优先级更高的中断请求，则优先处理此中断请求，

    • 中断屏蔽技术：

  • DMA方式：主存与I/O有直接的数据通路，因此传输数据时，无需调用中断服务程序

  • 通道方式：系统设有通道和通道控制部件，每个通道挂接若干外设，主机在执行I/O命令时，只需启动相关通道，通道将执行通道程序（存储在主存中），从而完成I/O
